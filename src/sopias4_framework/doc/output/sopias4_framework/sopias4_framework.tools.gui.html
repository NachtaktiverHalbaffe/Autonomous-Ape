sopias4_framework.tools.gui.gui_logger module

class sopias4_framework.tools.gui.gui_logger.GuiLogger(widget: QTextEdit, node: Node, fontSize: int = 12, *args, **kwargs)[source]

Bases: object

This class takes a QTextEdit element as a widget and displays the ros_messages in this element. For this purpose, it internally creates a subscription to the /rosout topic where all log messages are sent to. For this purpose, it is neccessary that the gui node is running. As a best practise, instantiate this logger inside the connect_labels_to_subscriptions() method.

Example:

class YourGuiImplementation(GUINode):
    def __init__(self) -> None:
        self.ui: Ui_MainWindow
        super().__init__(Ui_MainWindow())


    def connect_labels_to_subscriptions(self):
        # reference the desired QTextEdit element and the gui node here
        GuiLogger(widget=self.ui.textEdit, node=self.node)

This class takes a QLabel element as a widget and connects it to an ROS2 subscription to to display the content of its messages in the GUI

Example:

class YourGuiImplementation(GUINode):

    def __init__(self) -> None:
        self.ui: Ui_MainWindow
        super().__init__(Ui_MainWindow())

    def connect_labels_to_subscriptions(self):
        # reference the desired QLabel element, the gui node and the message type here
        LabelSubscriptionHandler(
            widget=self.ui.label_battery, node=self.node, message_type=BatteryState
        )
